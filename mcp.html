MCP Server

Model Context Protocol — connect Claude to external tools, APIs, and services. Plugs into the stack at any layer.

MCP Server connects Claude to the world outside the repo.
Available integrations
GitHub — repos, PRs, issues
Database — query, inspect schema
External APIs — REST, GraphQL
Custom tools — internal services
Slack — notifications, messages
Jira / Linear — issue tracking
Configure in .claude/settings.json
 .claude/settings.json
{ "mcp_servers": [ { "name": "github", "url": "https://mcp.github.com", "auth": { "type": "bearer", "token": "$GITHUB_TOKEN" } }, { "name": "my-db", "command": "npx @myorg/db-mcp-server", "env": { "DB_URL": "$DATABASE_URL" } } ], "hooks": { "PreToolUse": "hooks/PreToolUse.sh", "PostToolUse": "hooks/PostToolUse.sh", "SessionStart": "hooks/SessionStart.sh", "Stop": "hooks/Stop.sh", "SubagentStop": "hooks/SubagentStop.sh" } }
URL-based server
Remote MCP
Hosted endpoint, auth via bearer token
Good for SaaS tools (GitHub, Slack)
URL + token in settings.json
Command-based server
Local MCP
Spawns a local process via npx/node
Good for internal APIs and databases
Env vars passed via env block
MCP servers declared in manifest.json (L5) travel with your plugin — every teammate gets the same external tool connections automatically on install.
