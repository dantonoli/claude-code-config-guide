SKILL.md template
--- name: pdf-skill description: Use when reading, converting, or extracting content from PDF files. Handles text, tables, images, and form fields. --- ## Environment - Primary lib: pymupdf4llm (best for LLM pipelines) - Fallback: pdfplumber for table extraction - Output dir: /tmp/pdf-out/ ## Workflow 1. Read with pymupdf4llm.to_markdown(path) 2. For tables: pdfplumber.open(path).pages[n].extract_table() 3. Save output to /tmp/pdf-out/{filename}.md ## Never do - Never use PyPDF2 (deprecated) - Never load entire PDF into context if > 50 pages
The description field is how Claude decides to pull in the skill. Write it to match exactly how users phrase requests — "convert this PDF", "read this PDF", "extract tables from..."

Hooks — Guardrail Layer

Deterministic. Not AI. Shell scripts that fire on agent events. Git hooks, but for your agent.

Hooks turn vibes into rules. Git hooks, but for your agent.

Bash(rm *)
Wildcard matchers on tool name
Regex matchers on command

Command
if [ ... ]; then exit 2; fi
Block dangerous tools (exit 2)
Inject context (echo to stdout)
Audit log (append to file)
All hook types
PreToolUse.shInspect or block before any tool runs
PostToolUse.shLint, log, or notify after tool runs
SessionStart.shLoad context when a session begins
Stop.shRun when Claude finishes a turn
SubagentStop.shRun when a subagent returns
PreToolUse.sh — block dangerous commands
#!/bin/bash # hooks/PreToolUse.sh — exit 2 = block, exit 0 = allow TOOL="$1" INPUT=$(cat) # JSON piped to stdin if [[ "$TOOL" == "bash" ]]; then CMD=$(echo "$INPUT" | jq -r '.command') # Block rm -rf with wildcard matcher if echo "$CMD" | grep -qE 'rm\s+-rf'; then echo "BLOCKED: rm -rf not allowed" >&2 exit 2 # ← exit 2 = hard block fi # Block prod kubectl without --dry-run if echo "$CMD" | grep -q 'kubectl.*prod'; then if ! echo "$CMD" | grep -q '--dry-run'; then echo "BLOCKED: prod kubectl requires --dry-run" >&2 exit 2 fi fi fi exit 0 # allow
PostToolUse.sh — auto-lint + SubagentStop.sh
#!/bin/bash # hooks/PostToolUse.sh FILE=$(echo "$(cat)" | jq -r '.path // empty') if [[ "$1" == "write_file" ]] && [[ "$FILE" == *.py ]]; then ruff check --fix "$FILE" 2>/dev/null fi exit 0 --- # hooks/SubagentStop.sh — fires when any subagent finishes RESULT=$(cat) # subagent return value as JSON echo "[audit] subagent done: $(date)" >> .claude/audit.log exit 0
Flow
① Event fires
→
② Matcher checks
→
③ Command runs
Exit code 2 = block the tool call. Exit 0 = allow. Keep hooks fast — they're synchronous and block every tool call.

Subagent definition template
--- # subagents/code-reviewer.md name: code-reviewer description: Reviews code for bugs, style, security tools: [read_file, list_directory] # no write access model: claude-sonnet-4-20250514 --- ## Role Senior code reviewer. Receive a diff or file path. Return structured findings only — no chit-chat. ## Output format (return ONE message) { "issues": [ { "line": N, "severity": "high|med|low", "message": "..." } ] } ## Rules - Flag SQL injections, XSS, secrets in code - Check naming conventions from .claude/CLAUDE.md - Always return even if empty: { "issues": [] }
Agent Teams: run subagents in parallel via message passing with shared permissions. One per module — main session only sees aggregated results.
