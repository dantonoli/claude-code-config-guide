📁 agent-dev-kit/ Five layers, one stack
Layer 1 CLAUDE.md (The Memory Layer)
├📁 CLAUDE.md/ L1 . Memory Layer
│  ├▯ architecture.rules Naming, structure, repo expectations
│  ├▯ global.md Lives at ~/.claude/, every project
│  └▯ project.md Lives at .claude/, this repo only
Layer 2 Skills (The Knowledge Layer)
├📁 skills/ L2 . Knowledge Layer
│  ├▯ SKILL.md Description matched, auto invoked
│  ├▯ scripts/ Reference scripts and templates
│  └▯ context.md On demand, isolated subagent
Layer 3 Hooks (The Guardrail Layer)
├📁 hooks/ L3 . Guardrail Layer
│  ├▯ PreToolUse.sh Block rm -rf, validate before run
│  ├▯ PostToolUse.sh Auto lint on Write, ping Slack
│  ├▯ Stop.sh Run when Claude finishes a turn
│  └▯ SubagentStop.sh Deterministic. Not AI. Git hooks.
Layer 4 Subagents (The Delegation Layer)
├📁 subagents/ L4 . Delegation Layer
│  ├▯ code-reviewer.md Own context window, results only
│  ├▯ test-runner.md Custom tools, custom permissions
│  ├▯ explorer.md Keeps the main context clean
│  └▯ feature-dev.md Designs and implements end-to-end
Layer 5 Plugins (The Distribution Layer)
└📁 plugins/ L5 . Distribution Layer
   ├▯ manifest.json Bundles skills, agents, hooks, cmds
   ├▯ marketplace.url npm packages for agent capabilities
   └▯ team.install One install, every teammate aligned
